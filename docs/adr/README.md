# Architecture decision records

Architecture decision records (ADRs) capture decisions that future changes
must either follow or explicitly supersede.

- [ADR-0001: Separate institutional access from source selection](0001-institutional-access-boundaries.md)
- [ADR-0002: Resolve browser publisher pages through one deep module](0002-publisher-pdf-resolver.md)
- [ADR-0003: Keep institutional credentials local](0003-cookie-lifecycle-and-security.md)
- [ADR-0004: Publish under a downstream distribution identity](0004-release-identity-and-versioning.md)

Create the next record with a four-digit sequence number. Do not rewrite an
accepted decision after its consequences have shipped; add a superseding ADR.
